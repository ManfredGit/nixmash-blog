v1.0.0.0308

  • Twitter Card Data Creation complete.
  • New SQL table post_meta. PostMeta Entity added to Post Model with Fetch Eager
  • Three Card Options are: NONE, SUMMARY, and SUMMARY_LARGE_IMAGE
  • Several supporting properties in External blog.properties: twitterSite, twitterCreator and twitterImage.
  • The first post image is saved as the Twitter Card Image. The TwitterImage property is the fallback default card image when post contains no images.
  • For SINGLEPHOTO_POST types, the Post Image is the Card Image.
  • For MULTIPHOTO_POST types, the first of the post images is the Card Image.
  • TAGS: Realizing the [ENTER] Tag Entry key caused the Post Form Submit Event, new tags are now entered with [CTRL]-[ENTER]. A Help Glyphicon and Dialog added to the Tag Form Label.

v1.0.0.0306

  • Categories are complete with the following rules and features:
  • 1) One Category Per Post (unlike WordPress which supports multiple categories per post)
  • 2) Categories can be marked INACTIVE, which simply means they do not appear on Post Edit form category selection
  • 3) One category is marked DEFAULT which will appear first in category selection for ease of use
  • 4) All posts require an assigned category. When a category is deleted its assigned posts are given the UNCATEGORIZED category
  • 5) The UNCATEGORIZED category cannot be modified or removed
  • See this NixMash Post for more.
  • There are no plans for implementing a "Display Posts by Category" or similar features at this time. Categories were added primarily for completeness and because they are supported in WordPress

v1.0.0.0224

  • Permalink Post routing methods in a new PermalinkPostController class
  • The Permalink Post Url is in the format /post/{postname} (No category)
  • Details: Those are for users who migrate WordPress blogs with url /{category}/{postname}
  • Two methods, one with active categories with auto rerouting to /post/{postname}.
  • Second method throws an CategoryNotSupported Exception and displays a page with some verbage and link to the post
