v1.0.0.0321

Focusing on the New Blog experience

  • An exception was thrown when a post was created without first creating a Category. This is a little tricky since some users will use the WordPress NixMash Migrator app to import their WP Categories. Solution: Remind all users to run /install/sql/initialize.sql before firing up their site which creates the required Uncategorized Category if not already present.
  • And on New Blog, New Posts, an exception was thrown if Solr was not enabled and configured. A new External Property blog.solrEnabled in blog.properties was added to bypass indexing new posts and links. The setting is limited to post administration tasks, as it does not hide user Search fields or Search-related links.
  • A new nolinks.ftl FreeMarker template displays when no Links are available.
  • Duplicate Posts were retrieved and displayed. Problem was due to these posts being assigned multiple categories which were migrated from WordPress. Script was added to /install/sql/initialize.sql to remove multiple post category assignments in table post_category_ids.

v1.0.0.0306

  • Categories are complete with the following rules and features:
  • 1) One Category Per Post (unlike WordPress which supports multiple categories per post)
  • 2) Categories can be marked INACTIVE, which simply means they do not appear on Post Edit form category selection
  • 3) One category is marked DEFAULT which will appear first in category selection for ease of use
  • 4) All posts require an assigned category. When a category is deleted its assigned posts are given the UNCATEGORIZED category
  • 5) The UNCATEGORIZED category cannot be modified or removed
  • See this NixMash Post for more.
  • There are no plans for implementing a "Display Posts by Category" or similar features at this time. Categories were added primarily for completeness and because they are supported in WordPress
